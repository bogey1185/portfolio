My Projects


Farkle

This web-based adaptation of the classic dice game accommodates 2 to 8 players. In Farkle, players take turns rolling 6 dice to accumulate points. After each roll, a player gets to decide whether to continue rolling (and risk accumulated points) or play it safe! First player to 10,000 points wins!

This is the first application I created at General Assembly. It is a simple web app created entirely with HTML, CSS, and Javascript. The simple UI of this app is made possible by complicated scoring algorithms operating under-the-hood. Check out the code here!



Woodworkers' Resource Center

WWRC is a website created by woodworkers for woodworkers. On WWRC, users can log in to create custom how-to guides for various woodworking projects. Projects are separated automatically into different categories based on project difficulty.

This is a full-stack NodeJS application built on the Express.js MVC framework. The app uses EJS templating on the front-end. On the back-end, the app uses Mongoose for modeling and MongoDB (NoSQL) for data storage. To learn more about this project, check out the code here!



LeaseGuard

LeaseGuard is a web application that allows tenants to smoothly document the state of their apartment at the beginning of a lease. The process begins with the landlord. The landlord creates an account for the property in question. LeaseGuard utilizes a 3rd party API to process the submitted property address and ensure its accuracy. Once the property account is created, the landlord can share a randomized hash code for the tenant to access the property. Finally, the tenant can log in, and upload photographs with accompanying text regarding the condition of the property.

This is a decoupled Javascript-based application. The front-end is built with React.js. The back-end is a Node.js API built with Express.js. The API communicates with a MongoDB (NoSQL) database via the Mongoose ODM. To learn more about this project, check out the React code, and the API code. Please note, this project is depoloyed via Heroku. Heroku servers go to "sleep" when not in use, so it may take a few moments to load the webpage!



Pollster

Pollster is a fun web application that allows users to create public or private polls. See what the public things about a question, or share a private poll with your friends!

Pollster is a full-stack Python application utilizing Python Flask. Flask is a light-weight MVC framework similar to Express.js. The app uses Jinja2 templating on the front-end. The back-end uses the Peewee3 ORM for communication with a PostgreSQL database. To learn more about this project, check out the code here! Please note, this project is depoloyed via Heroku. Heroku servers go to "sleep" when not in use, so it may take a few moments to load the webpage!



CrowdTales

CrowdTales is an application that combines creative writing and crowd sourcing. In the app, users can start a new story by creating a writing prompt. Thereafter, users can sign up to help create content for the story. Stories are written one chapter at a time!

CrowdTales is my capstone project from my time at General Assembly. CrowdTales is a decoupled application. While, the front-end is built with React.js, I opted to build a Python Flask API for the back-end. Additionally, since this project required a lot of relational data, I chose to utilize a PostgreSQL database managed by the Peewee3 ORM. To learn more about this project, check out the React code, and the API code. Please note, this project is depoloyed via Heroku. Heroku servers go to "sleep" when not in use, so it may take a few moments to load the webpage!
